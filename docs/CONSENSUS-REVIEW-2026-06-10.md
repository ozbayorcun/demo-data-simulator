# Consensus Review - 2026-06-10

## Scope

One final pre-public review pass split into two tracks:

- implementation/code quality
- README/docs/open-source readiness

## Consensus Verdict

The concept is strong, but the repo was not public-ready before this pass. Reviewers converged on two themes:

- safety hardening was needed before publishing a CLI that reads repos and deletes/recreates output directories
- npm package contents and README examples needed to match actual shipped behavior

## Findings Accepted And Fixed

- Guarded `generate --out` so it refuses filesystem root, current working directory, and non-empty directories not previously generated by this tool.
- Validated `--max-files` and `--max-bytes` as positive integers so bad values cannot disable evidence caps.
- Expanded secret redaction for bearer authorization headers, database URLs, URL credentials, and common token/secret key names.
- Added spec validation for `defaults.days`, `defaults.startDate`, and numeric field bounds.
- Made `outputs.formats` control which output families are written.
- Made missing custom agent binaries return a clean inference error instead of crashing.
- Removed misleading forward-reference warnings for valid `ref:` fields.
- Added `prepack` so npm packaging rebuilds `dist`.
- Stopped shipping internal planning docs in the npm package.
- Included runnable example fixtures in the npm package.
- Clarified README claims around Claude readiness, manual mode, reproducibility, and custom-agent trust.
- Added status notes to planning/evaluation docs to distinguish roadmap/history from shipped user docs.

## Verification

- `npm run check`
- `npm pack --dry-run`
- CLI smoke: invalid evidence limits fail closed
- CLI smoke: unsafe output directories fail closed
- CLI smoke: missing custom agent binary returns an inference error
- CLI smoke: command-agent field-service inference still works
